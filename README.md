# `Photos_Safe.app` 導入＆引き継ぎマニュアル

`Photos_Safe.app` は、Macの「写真」アプリを安全に起動するためのランチャーアプリです。
外付けドライブに写真ライブラリを保存している場合、ドライブ未接続時に写真アプリを開くと、Mac本体に勝手に新しいライブラリが作られてしまう事故を防ぐことができます。

---

## 📖 インストール・設定手順

別のMacでこの `Photos_Safe.app` を使うための手順です。

### 0. 実行権限の付与と検疫の解除（初回のみ）
Gitからクローンやダウンロードした場合、アプリの実行権限が外れ、Macのセキュリティ制限（検疫）がかかっていることがあります。
まずはこれらを解除します。

ターミナルを開き、本リポジトリのディレクトリで以下のコマンドを実行してください。

```bash
./fix_app_permissions.sh Photos_Safe.app
```

> **注意:** 実行権限がないと言われた場合は、先にスクリプトに権限を付与してください。
> `chmod +x fix_app_permissions.sh`

### 1. アプリの配置
1. `Photos_Safe.app` を、Macの **「アプリケーション」フォルダ** に移動（コピー）させます。

### 2. Dock（下のバー）への登録
このアプリを普段使う写真アプリの代わりに設定します。

1. 画面下のDockにある**元々の「写真」アプリ**のアイコンを右クリックします。
2. 「オプション」＞「**Dockから削除**」をクリックして消します。（※アプリ自体は消えません）
3. 最初に「アプリケーション」フォルダに入れた `Photos_Safe.app` を、そのままDockに向かってドラッグ＆ドロップし、追加します。

### 3. 初期設定（パスの書き換え）
もし、新しいMacに繋ぐ外付けSSDの名前がデフォルトと違う場合や、写真ライブラリの場所が異なる場合は、中の設定を書き換える必要があります。

1. Macの「アプリケーション」フォルダから「**Automator**（ロボットがパイプを持っているアイコン）」を開きます。
2. 画面が出たら「完了」や「新規書類」ではなく、**「既存の書類を開く」**（またはメニューの「ファイル」＞「開く」）を選びます。
3. Dockに入れた `Photos_Safe.app` （または「アプリケーション」フォルダ内の `Photos_Safe.app`）を選んで開きます。
4. 画面の右側に、シェルスクリプトが表示されます。
5. 上のほうにある2行目を、**ご自身のSSDの実際の名前に書き換えます**。
   ```bash
   # 例：SSDの名前が "Backup_Drive" の場合
   VOLUME="/Volumes/Backup_Drive"
   ```
6. 3行目の写真ライブラリまでのフォルダ階層も、実際の場所に書き換えます。
   ```bash
   LIBRARY="$VOLUME/あなたのフォルダ/Photos Library.photoslibrary"
   ```
7. 書き換えたら、キーボードの `Command` と `S` を同時に押して保存し、Automatorを終了します。

### 4. 使い方
普段通り、Dockに追加した `Photos_Safe.app` のアイコンをクリックして写真を開くだけです。

* **SSDが繋がっている時**：何も起きず、普通に写真アプリが開きます。
* **SSDが繋がっていない時**：「写真ライブラリが見つかりません」という警告が出ます。
  * `【終了】` を押せば、そのまま何も開かずに安全に終了します。（勝手に本体にライブラリが作られることはありません）
  * `【新たに指定】` を押せば、画面から「本当はここにある」という別の `.photoslibrary` ファイルを選んで開くことができます！新たに指定した場合、次回以降のデフォルトになるようにシステム環境設定での変更を促す通知が出ます。

## 👨‍💻 技術的な仕組み (Handoff)

このアプリはmacOS標準の「Automator」を使用して作成された「**アプリケーション**」形式のファイルです。
内部でシェルスクリプト（zsh）とAppleScriptを実行しています。

### 動作フロー
1. 起動すると、設定されたパス（デフォルトのドライブおよび写真ライブラリ）の存在を確認します。
2. **ライブラリが見つかった場合**:
   - そのまま `open -a "Photos" [ライブラリのパス]` で写真アプリを起動し、正常終了します。
3. **ライブラリが見つからなかった場合**:
   - `osascript` (AppleScript) を用いて警告ダイアログを表示します。
   - 「終了」または「新たに指定」の選択肢を提示します。
4. **「新たに指定」が選ばれた場合**:
   - AppleScriptの `choose file` ダイアログを開き、ユーザーに任意の `.photoslibrary` ファイルを選択させます。
   - 選択されたライブラリパスを使用して写真アプリを起動します。
   - 同時に「次回からこれをデフォルトにするには設定を変更してください」というシステム通知を表示します。
